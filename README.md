## Description

API full REST built with [Nest](https://github.com/nestjs/nest)

The idea is to manage multiple functionalities against different personal projects to apply the best practices in backend (SOLID, DRY, etc)

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ docker compose -f ./docker-compose.dev.yml up
```

If you want to develop full local you can change

```
MONGO_DB_CLUSTER_URL=mongodb://mongodb-hz-api:27017/hs-server-api
```


```
MONGO_DB_CLUSTER_URL=mongodb://127.0.0.1:27017/hs-server-api 
```

in this way, you can run the mongodb service and to be recognized by the local machine


## Run hz-server-api / mongodb database with Docker Compose
URL generated by the mongodb container

```http://localhost:3001/api/tasks ```

## Swagger API Docs

Use the following url in local to access swagger docs

```http://localhost:3001/api/docs#/```



## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```
